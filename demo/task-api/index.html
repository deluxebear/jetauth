Task API Demo

A tiny service for exercising JetAuth WAF URL-level authorization. Deploy behind a site with EnableBizAuthz on, then grant different subsets of the endpoints below to different roles.

Endpoints

OpenAPI 3.1 spec

Import /openapi.yaml into Application Authorization → Resources → Import → OpenAPI. Tags become groups; operationId becomes the resource name so re-imports update existing rows instead of creating duplicates.

Try it

curl http://localhost:8081/api/health
curl http://localhost:8081/api/me
curl http://localhost:8081/api/tasks

When the request bypasses the WAF, /api/me returns an empty identity with a note. When routed through the WAF, you will see the X-Forwarded-User / -Email headers the gateway injected after SSO.
