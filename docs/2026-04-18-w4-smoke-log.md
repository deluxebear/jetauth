## Manual browser checklist

With the dev server from W3 still running, test these in the admin UI:

1. Open any app's "界面定制" tab → preview iframe appears on the right (40/60 split)
2. Change `themeData.colorPrimary` via ColorPicker palette → preview recolors
3. Drag Password row up/down in "登录方式" table → iframe reorders
4. Drag "Back button" visibility toggle in "登录项" table → preview shows/hides back chip
5. Click "Mobile" in PreviewToolbar → iframe shrinks to 375px centered with rounded bezel
6. Click "Dark" → iframe reloads with dark theme param (requires W6's runtime dark toggle; for W4 we just pass the param)
7. Click "Signup" mode → iframe navigates to /signup/:app
8. Resize window below 1280px → preview collapses to "Show live preview" button

## Known limitations (deferred to W5/W6)

- **Dark mode toggle:** preview `?previewTheme=dark` is wired, but AuthShell currently doesn't react to it (dark theme in the auth UI is a separate system toggle). Full wiring is W5/W6.
- **HTML injection preview:** W5 adds the actual `headerHtml`/`footerHtml`/`signinHtml`/`signupHtml` rendering. W4 carries their form-state values into the preview config but they're not yet rendered.
- **postMessage incremental updates:** currently forces iframe re-create via `key={src}` change. Works but causes a brief blank frame. W6 polish.
- **Drag-sort accessibility:** keyboard support works via @dnd-kit defaults (arrow keys while focused on handle). Screen-reader announcements via aria-live are @dnd-kit's built-in; we haven't audited them.
