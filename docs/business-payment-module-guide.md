# 商业 & 付款模块 — 产品说明

## 模块概述

商业 & 付款模块为 JetAuth 提供完整的电商和订阅付费能力。涵盖商品管理、购物车、下单支付、订阅管理和交易记录全链路。

| 模块 | 导航路径 | 功能定位 |
|------|---------|---------|
| 商品商店 | 商业 & 付款 → 商品商店 | 面向用户的商品展示和购买入口 |
| 商品 | 商业 & 付款 → 商品 | 管理员维护商品信息（价格/库存/支付方式） |
| 购物车 | 商业 & 付款 → 购物车 | 查看和管理已加入购物车的商品 |
| 订单 | 商业 & 付款 → 订单 | 查看和管理所有订单 |
| 付款 | 商业 & 付款 → 付款 | 查看支付记录和发票信息 |
| 计划 | 商业 & 付款 → 计划 | 定义订阅计划（月付/年付/权限角色） |
| 定价 | 商业 & 付款 → 定价 | 将多个计划组合为定价方案 |
| 订阅 | 商业 & 付款 → 订阅 | 查看用户的订阅状态 |
| 交易 | 商业 & 付款 → 交易 | 查看所有支付和充值的交易记录 |

---

## 核心业务流程

### 购买流程

```
商品商店 → 选择商品 → 购买页（选择数量/充值金额）
  ├→ 立即购买 → 订单支付页（选择支付方式）→ 支付 → 支付结果页
  └→ 加入购物车 → 购物车页 → 结算 → 订单支付页 → 支付 → 支付结果页
```

### 订阅流程

```
定价方案 → 选择计划 → 下单 → 支付 → 订阅自动激活 → 到期后自动变为已过期
```

### 充值流程

```
充值商品 → 选择金额（预设或自定义）→ 下单 → 支付 → 用户余额增加
```

## 一、商品商店

### 功能说明

商品商店是面向终端用户的商品展示页面，以卡片网格布局展示所有已上架（Published）的商品。支持搜索和按标签筛选。

### 页面功能

- **商品卡片**：展示商品图片、名称、描述、标签、价格（或充值选项）、已售数量
- **搜索**：按商品名称、显示名称、描述、标签进行实时搜索
- **标签筛选**：自动收集已上架商品的标签，点击筛选同标签商品
- **充值商品标识**：充值类商品显示蓝色"充值"标签
- **售罄标识**：库存为 0 的商品显示红色"已售罄"标签

### 操作指南

1. 进入 **商业 & 付款 → 商品商店**
2. 浏览商品卡片，使用搜索框或标签筛选缩小范围
3. 点击商品卡片进入购买详情页

## 二、商品管理

### 功能说明

管理员创建和维护商品信息，包括基本信息、图片、定价、支付提供商和状态。

### 字段说明

| 字段 | 说明 |
|------|------|
| 组织 | 商品所属的组织 |
| 名称 | 唯一标识名，系统自动生成 |
| 显示名称 | 面向用户的可读名称 |
| 标签 | 商品分类标签（用于商店筛选） |
| 描述 | 简短描述（200 字以内） |
| 详情 | 详细描述（1000 字以内） |
| 图片地址 | 商品展示图片 URL |
| 货币 | 商品使用的货币（如 USD、CNY），选定后建议不修改 |
| 充值商品 | 开启后商品变为充值类型，支付成功后增加用户余额 |
| 禁用自定义金额 | 仅充值商品可见，开启后用户只能选择预设金额 |
| 充值金额选项 | 仅充值商品可见，预设的充值金额列表 |
| 价格 | 仅非充值商品可见，商品固定价格 |
| 库存 | 剩余可售数量（充值商品忽略此字段） |
| 已售 | 已售出数量（系统自动维护） |
| 支付提供商 | 商品支持的支付方式（如 Alipay、Stripe、Balance） |
| 成功跳转地址 | 支付成功后的回调 URL |
| 状态 | Published（已上架）或 Draft（草稿），仅 Published 商品在商店可见 |

### 操作指南

1. 进入 **商业 & 付款 → 商品**，点击「添加商品」
2. 填写基本信息（名称、显示名称、标签、描述）
3. 上传商品图片（填入图片 URL）
4. 设置定价：
   - **普通商品**：选择货币，填入价格和库存
   - **充值商品**：开启"充值商品"，添加充值金额选项
5. 选择支付提供商（需先在 **认证 → 提供商** 中配置 Payment 类型的提供商）
6. 状态设为 Published 后商品在商店可见
7. 点击「保存」

### 注意事项

- 同一订单内的所有商品必须使用相同货币
- 支付宝（Alipay）仅支持 CNY
- 充值商品不检查库存，可无限售卖
- 删除商品前确保没有未完成的订单引用该商品

## 三、购买页

### 功能说明

商品购买详情页展示单个商品的完整信息，支持选择数量或充值金额，提供"立即购买"和"加入购物车"两种操作。

### 操作指南

#### 购买普通商品

1. 从商品商店点击商品进入购买页
2. 使用数量选择器调整购买数量（默认 1）
3. 页面显示小计金额
4. 点击「立即购买」→ 系统创建订单 → 自动跳转到支付页
5. 或点击「加入购物车」→ 商品加入购物车，可稍后结算

#### 购买充值商品

1. 进入充值商品购买页
2. 选择预设金额（如 ¥10、¥50、¥100）或输入自定义金额
3. 点击「立即购买」→ 系统创建订单 → 跳转支付页
4. 充值商品**不支持加入购物车**，必须直接购买

### 注意事项

- 加入购物车时会检查货币一致性，不同币种的商品不能同时在购物车中
- 库存为 0 的商品"立即购买"和"加入购物车"按钮均被禁用

## 四、购物车

### 功能说明

查看已加入购物车的商品，支持修改数量、删除商品、清空购物车和批量结算。

### 页面功能

- **商品列表**：显示图片、名称、单价、数量、小计
- **数量编辑**：通过加减按钮或直接输入调整数量
- **删除商品**：点击删除图标移除单个商品
- **清空购物车**：一键清空所有商品（需确认）
- **无效商品检测**：已下架或不存在的商品标红显示，不参与结算
- **合计与结算**：底部固定栏显示总价和商品数量，点击「结算」下单

### 操作指南

1. 进入 **商业 & 付款 → 购物车**
2. 检查商品和数量，按需调整
3. 点击「结算」→ 系统创建订单（包含购物车所有有效商品）→ 自动清空购物车 → 跳转支付页

### 数据存储

购物车数据存储在用户（User）对象的 `cart` 字段中，跨设备同步。

## 五、订单支付页

### 功能说明

展示订单详情并选择支付方式完成支付。根据订单状态自动切换"支付模式"和"查看模式"。

### 页面功能

- **订单信息**：订单号、创建时间、状态、商品列表（名称 × 数量 = 小计）、总金额
- **支付方式**：从商品关联的支付提供商列表加载可用支付方式，显示为可选卡片
- **确认支付**：选择支付方式后点击「支付」按钮
- **取消订单**：点击「取消」放弃此订单

### 支付方式处理

| 支付方式 | 行为 |
|---------|------|
| Balance（余额） | 直接从用户余额扣款，立即完成，跳转支付结果页 |
| Alipay（支付宝） | 跳转支付宝收银台页面 |
| WeChat Pay（微信支付） | 微信内调用 JSAPI 支付，其他环境显示二维码 |
| Stripe | 跳转 Stripe Checkout 页面 |
| PayPal | 跳转 PayPal 支付页面 |
| 其他提供商 | 跳转对应提供商的支付页面 |

### 订单状态

| 状态 | 说明 | 可操作 |
|------|------|--------|
| Created | 已创建，等待支付 | 可支付、可取消 |
| Paid | 已支付 | 只读，可查看支付详情 |
| Canceled | 已取消 | 只读 |
| Failed | 支付失败 | 只读 |
| Timeout | 支付超时 | 只读 |

## 六、支付结果页

### 功能说明

支付完成后展示支付结果，支持自动轮询等待异步支付回调。

### 状态展示

| 状态 | 图标 | 说明 |
|------|------|------|
| Paid | 绿色勾 | 支付成功，显示金额和商品信息 |
| Created | 加载动画 | 处理中，每 2 秒自动检查支付状态 |
| Canceled | 黄色警告 | 支付已取消 |
| Timeout | 灰色时钟 | 支付超时 |
| Error | 红色叉 | 支付失败，显示失败原因 |

### 操作

- **查看订单**：跳转到对应订单详情页
- **返回商店**：返回商品商店继续购物

## 七、计划管理

### 功能说明

定义订阅计划（Plan），每个计划有固定的价格、周期和关联角色。创建计划时系统会自动生成一个对应的虚拟商品。

### 字段说明

| 字段 | 说明 |
|------|------|
| 名称 | 计划唯一标识 |
| 显示名称 | 面向用户的计划名称 |
| 价格 | 计划周期价格 |
| 货币 | 计划使用的货币 |
| 周期 | Monthly（月付）或 Yearly（年付） |
| 是否排他 | 开启后用户最多只能有一个活跃订阅 |
| 角色 | 订阅激活后自动授予用户的权限角色 |
| 支付提供商 | 计划支持的支付方式 |

### 操作指南

1. 进入 **商业 & 付款 → 计划**，点击「添加计划」
2. 填写名称、价格、周期、支付方式
3. 选择关联角色（可选，订阅激活后用户自动获得此角色的权限）
4. 如果是排他计划，开启"是否排他"

## 八、定价管理

### 功能说明

将多个计划组合为一个定价方案（Pricing），关联到应用。用户通过定价方案选择适合的计划进行订阅。

### 字段说明

| 字段 | 说明 |
|------|------|
| 名称 | 定价方案唯一标识 |
| 应用 | 关联的 JetAuth 应用 |
| 计划列表 | 包含的计划（如 Basic、Pro、Enterprise） |
| 试用天数 | 试用期时长（0 表示无试用） |
| 是否启用 | 启用后定价方案可被前端使用 |

---

## 九、订阅管理

### 功能说明

查看和管理用户的订阅状态。订阅在用户购买包含计划的订单并完成支付后自动创建和激活。

### 订阅状态

| 状态 | 说明 |
|------|------|
| Pending | 等待支付完成 |
| Active | 活跃中（在有效期内） |
| Upcoming | 已购买但还未到开始时间 |
| Expired | 已过期 |
| Suspended | 被管理员暂停 |
| Error | 关联支付出错 |

### 自动状态更新

每次查询订阅列表时，系统自动根据当前时间和支付状态更新订阅状态：
- 未到开始时间 → Pending/Upcoming
- 超过结束时间 → Expired
- 在有效期内 → Active
- 关联支付失败 → Error

## 十、交易记录

### 功能说明

记录所有支付和充值的交易明细，用于账务审计。

### 交易类型

| 类型 | 金额方向 | 说明 |
|------|---------|------|
| Purchase（购买） | 负数 | 用户购买商品的支出记录 |
| Recharge（充值） | 正数 | 用户充值余额的收入记录 |

### 字段说明

| 字段 | 说明 |
|------|------|
| 类别 | Purchase 或 Recharge |
| 金额 | 交易金额（正数为收入，负数为支出） |
| 货币 | 交易使用的货币 |
| 用户 | 交易关联的用户 |
| 提供商 | 支付提供商名称 |
| 支付单号 | 关联的支付记录 |
| 状态 | Paid 等 |

## 十一、付款记录

### 功能说明

查看所有支付记录，包括支付状态、支付方式、金额、发票信息等。

### 发票功能

支付成功后可申请发票：

1. 进入付款记录，找到状态为 Paid 的支付记录
2. 编辑支付记录，填写发票信息（类型、抬头、税号、备注）
3. 系统通过支付提供商获取发票 URL

---

## 支付提供商配置

商品和计划的支付方式依赖于已配置的 Payment 类型提供商。

### 配置步骤

1. 进入 **认证 → 提供商**，点击「添加提供商」
2. 类别选择 **Payment**
3. 类型选择支付渠道（Alipay / WeChat Pay / Stripe / PayPal / Balance 等）
4. 填写对应渠道的 Client ID 和 Client Secret
5. 保存后，在商品或计划的"支付提供商"字段中可选择此提供商

### 支持的支付渠道

| 渠道 | 支持币种 | 备注 |
|------|---------|------|
| Alipay | 仅 CNY | 国内支付 |
| WeChat Pay | 多币种 | 需配置微信开放平台 |
| Stripe | 多币种 | 国际信用卡 |
| PayPal | 多币种 | 国际支付 |
| Balance | 多币种 | 用户账户余额，即时到账 |
| Airwallex | 多币种 | 跨境支付 |

### Balance（余额支付）

Balance 是内置的钱包支付方式，使用用户账户余额直接扣款：
- 无需跳转第三方页面，即时完成
- 余额通过充值商品增加
- 余额按货币分别管理（如 USD 余额和 CNY 余额独立）

## 关键业务规则

| 规则 | 说明 |
|------|------|
| 货币一致性 | 同一订单内所有商品必须使用相同货币 |
| 库存管理 | 普通商品在支付成功后扣减库存，充值商品不扣减 |
| 排他计划 | 标记为排他的计划，同一用户最多有一个活跃订阅 |
| 支付幂等 | 同一支付记录不会被重复处理 |
| 快照机制 | 下单时保存商品信息快照，后续商品变更不影响已有订单 |
| 购物车持久化 | 购物车存储在用户对象中，跨设备同步 |
| 充值商品限制 | 充值商品不能加入购物车，必须在购买页直接购买 |
