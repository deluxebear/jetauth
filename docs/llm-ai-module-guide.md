# LLM AI 模块 — 产品说明

## 模块概述

LLM AI 模块为 JetAuth 提供 AI 智能体管理、MCP 工具代理、网关防护和日志追踪能力。包含 6 个子模块：

| 模块 | 导航路径 | 功能定位 |
|------|---------|---------|
| 智能体 | LLM AI → 智能体 | 管理 AI Agent 连接信息 |
| MCP 服务器 | LLM AI → MCP 服务器 | 代理和控制 MCP 工具调用 |
| MCP 商店 | LLM AI → MCP 商店 | 浏览和安装在线 MCP 服务器 |
| 入口 | LLM AI → 入口 | 查看系统日志和 AI 行为追踪 |
| 站点 | LLM AI → 站点 | 反向代理网关 + WAF 防火墙 |
| 规则 | LLM AI → 规则 | WAF/IP/UA/限速/组合访问控制规则 |

---

## 一、智能体管理

### 功能说明

智能体模块用于注册外部 AI Agent 的连接信息，包括访问地址和认证令牌。

### 字段说明

| 字段 | 说明 |
|------|------|
| 组织 | 智能体所属的组织（管理员可切换） |
| 名称 | 唯一标识名，系统自动生成，可修改 |
| 显示名称 | 可读名称，用于界面展示 |
| 监听地址 | Agent 的 HTTP/HTTPS 访问地址 |
| 访问令牌 | 用于认证的 Token（密码字段，保存后不可查看） |
| 应用 | 关联的 JetAuth 应用（按组织动态加载） |

### 操作指南

1. 进入 **LLM AI → 智能体**，点击「添加智能体」
2. 填写监听地址（如 `https://agent.example.com`）和访问令牌
3. 选择关联的应用（可选）
4. 点击「保存」或「保存并退出」

> **注意**：当前版本中智能体为预留功能，数据仅存储，尚未被其他模块消费。

## 二、MCP 服务器管理

### 功能说明

MCP（Model Context Protocol）服务器模块是 AI 工具调用的**代理中心**。它连接上游 MCP 服务器，同步可用工具列表，并提供工具级别的权限控制。外部 AI 通过 JetAuth 代理访问 MCP 工具时，系统会检查每个工具的 `IsAllowed` 状态。

### 核心流程

```
AI Agent → JetAuth 代理 (/api/server/:owner/:name)
         → 检查工具是否 isAllowed
         → 转发到上游 MCP 服务器
         → 返回工具执行结果
```

### 字段说明

| 字段 | 说明 |
|------|------|
| 组织 | 服务器所属组织 |
| 名称 | 唯一标识名 |
| 显示名称 | 可读名称 |
| URL | 上游 MCP 服务器地址（如 `https://mcp.example.com/sse`） |
| 访问令牌 | 连接上游服务器的认证 Token |
| 应用 | 关联的 JetAuth 应用 |
| 工具列表 | 从上游同步的工具，每个工具有独立的「允许」开关 |
| 基础地址 | 通过 JetAuth 代理访问此服务器的 URL（只读，自动生成） |

### 操作指南

#### 手动添加 MCP 服务器

1. 进入 **LLM AI → MCP 服务器**，点击「添加服务器」
2. 填写上游 MCP 服务器的 URL 和访问令牌
3. 保存后，点击「同步工具」按钮从上游获取可用工具列表
4. 在工具列表中，通过开关控制每个工具是否允许 AI 调用
5. 保存配置

#### 从 MCP 商店安装

1. 点击列表页的「MCP 商店」按钮
2. 浏览或搜索在线 MCP 服务器
3. 点击卡片上的「添加」按钮，自动创建服务器并跳转编辑页
4. 配置访问令牌后同步工具

#### 扫描局域网 MCP 服务器

1. 点击列表页的「扫描服务器」按钮
2. 配置扫描参数：
   - **IP 范围**：如 `192.168.1.0/24`（支持 CIDR）
   - **端口**：如 `3000`, `8080`, `1-65535`
   - **路径**：如 `/`, `/mcp`, `/sse`, `/mcp/sse`
3. 点击「开始扫描」，系统并发探测所有主机
4. 在结果表格中，点击「添加」将发现的服务器注册到系统

#### 工具权限管理

同步工具后，编辑页的「工具」区域显示所有可用工具的表格：

| 列 | 说明 |
|----|------|
| 名称 | 工具名（如 `get_weather`、`search_web`） |
| 描述 | 工具功能说明 |
| 允许 | 开关控制 — 关闭后 AI 调用该工具会被拒绝（返回 `tool is forbidden`） |

#### 基础地址的使用

保存服务器后，编辑页底部显示「基础地址」，格式为：

```
https://your-jetauth.com/api/server/{owner}/{name}
```

AI Agent 应使用此地址（而非上游直连地址）来调用 MCP 工具，这样所有工具调用都经过 JetAuth 的权限检查。

## 三、MCP 商店

### 功能说明

MCP 商店是一个在线 MCP 服务器目录，数据来源于 Casdoor 官方维护的注册表（`mcp.casdoor.org/registry.json`，已内置到系统中）。

### 操作指南

1. 进入 **LLM AI → MCP 服务器**，点击「MCP 商店」按钮
2. 浏览服务器卡片，每张卡片显示：
   - 服务器名称和描述
   - 生产端点 URL
   - 官方网站链接
   - 分类标签
3. 使用搜索框按名称筛选，或点击分类标签过滤
4. 点击「添加」一键创建 MCP 服务器实例

## 四、入口管理（日志追踪）

### 功能说明

入口模块是系统的**日志中心**，记录来自各类日志提供者的事件数据。入口记录由系统自动写入，不需要手动创建。

### 数据来源

| 来源 | 说明 |
|------|------|
| 权限日志提供者 | 每次权限检查时自动记录 |
| OpenClaw AI Agent | Agent 执行过程中实时推送行为数据（工具调用、结果、错误） |
| 外部日志采集器 | 通过 Provider 配置的采集器（如 SELinux Log）持续监听并写入 |

### 字段说明

| 字段 | 说明 | 可编辑 |
|------|------|--------|
| 组织 | 所属组织 | 管理员可改 |
| 名称 | 唯一标识（系统自动生成的十六进制字符串） | 可改 |
| 显示名称 | 可读名称 | 可改 |
| 提供商 | 日志来源（链接到提供商详情页） | 只读 |
| 应用 | 关联应用（链接到应用详情页） | 只读 |
| 类型 | 日志类型（如 `trace`、`session`） | 只读 |
| 客户端 IP | 请求来源 IP（可点击查询 IP 信息） | 只读 |
| 用户代理 | 请求的 User-Agent | 只读 |
| 消息 | 日志内容（JSON 格式，自动美化显示） | 只读 |

### 操作指南

1. 进入 **LLM AI → 入口**，查看日志列表
2. 列表支持按组织、名称、提供商、类型、客户端 IP、User-Agent 搜索和排序
3. 点击条目名称进入详情页，查看完整的消息内容
4. 客户端 IP 旁的链接按钮可跳转到 db-ip.com 查询 IP 地理位置信息

## 五、站点管理（反向代理 + WAF）

### 功能说明

站点模块是 JetAuth 内置的**反向代理网关和 Web 应用防火墙（WAF）**。它拦截所有经过的 HTTP/HTTPS 流量，按域名匹配站点配置，执行安全规则，然后将合法请求代理到后端服务。

### 核心架构

```
客户端请求 https://erp.company.com/api/orders
    ↓
JetAuth 网关 (端口 80/443)
    ↓
① 域名匹配 → 找到站点配置
② HTTP→HTTPS 重定向（如果 SSL 模式为 HTTPS Only）
③ OAuth 登录检查（如果配置了 JetAuth 应用）
④ 规则检查（WAF/IP/UA/限速规则，按顺序执行）
    ├─ 允许 → 继续
    ├─ 阻止 → 返回 403
    └─ 丢弃 → 返回 400
⑤ 反向代理 → 转发到后端服务器
```

### 字段说明

#### 基本信息

| 字段 | 说明 |
|------|------|
| 组织 | 站点所属组织 |
| 名称 | 唯一标识名 |
| 显示名称 | 可读名称 |
| 标签 | 分类标签（用于分组管理） |

#### 域名配置

| 字段 | 说明 |
|------|------|
| 域名 | 主域名（用于网关匹配，如 `erp.company.com`） |
| 其他域名 | 备用域名列表（也能匹配到此站点） |
| 需要重定向 | 开启后，通过备用域名访问时 301 重定向到主域名 |
| 禁用详细日志 | 开启后，被规则拦截时不在错误页显示规则详情 |

#### 规则

从规则模块选择要关联的规则，请求按列表顺序依次检查。

#### 告警配置

| 字段 | 说明 |
|------|------|
| 启用告警 | 开启后端健康检查 |
| 告警间隔 | 健康检查周期（秒），默认 60 |
| 告警重试次数 | 连续失败几次后触发告警，默认 3 |
| 告警提供商 | 通知渠道（邮件/短信提供商名称） |

#### 网络

| 字段 | 说明 |
|------|------|
| 主机 | 后端服务器地址（如 `192.168.1.100`） |
| 端口 | 后端服务器端口（如 `8080`） |
| 主机列表 | 多后端地址（随机负载均衡，优先于单主机设置） |
| 公网 IP | 域名解析到的公网 IP（系统自动检测，只读） |

#### SSL 配置

| 字段 | 说明 |
|------|------|
| SSL 模式 | `HTTP` / `HTTPS and HTTP` / `HTTPS Only` / `Static Folder` |
| SSL 证书 | 绑定的 TLS 证书（从证书管理模块选择） |

#### 状态

| 字段 | 说明 |
|------|------|
| JetAuth 应用 | 关联的 OAuth 应用（配置后，未登录用户自动跳转登录页） |
| 状态 | `Active`（启用）/ `Inactive`（停用） |

### 使用场景

#### 场景 1：给内部应用加 HTTPS + WAF 防护

```
站点配置：
  域名: erp.company.com
  主机: 192.168.1.100
  端口: 8080
  SSL 模式: HTTPS Only
  规则: [waf-sqli-rule, ip-whitelist-rule]

DNS: erp.company.com → JetAuth 服务器公网 IP

效果：
  外部访问 → HTTPS 加密 → WAF 检测 SQL 注入 → IP 白名单检查 → 代理到内网
```

#### 场景 2：给 SaaS 应用加 OAuth 登录保护

```
站点配置：
  域名: app.saas.com
  主机: localhost:3000
  JetAuth 应用: saas-app

效果：
  未登录 → 自动跳转 JetAuth 登录页 → 登录后跳回 → 代理到后端
```

#### 场景 3：多后端负载均衡

```
站点配置：
  域名: api.company.com
  主机列表: [10.0.0.1:8080, 10.0.0.2:8080, 10.0.0.3:8080]

效果：
  每次请求随机分配到 3 台后端之一
```

### 网关启动条件

网关仅在数据库中存在至少一条 Site 记录时启动，监听端口可在 `conf/app.conf` 中配置：

```
gatewayHttpPort = 80
gatewayHttpsPort = 443
```

### 后台自动行为

- 每 5 秒刷新站点和规则缓存
- 自动检查 SSL 证书过期，通过 Let's Encrypt 自动续期

## 六、规则管理（WAF / 访问控制）

### 功能说明

规则模块定义站点的访问控制策略。每条规则包含一组表达式，匹配后执行允许或阻止动作。规则通过站点模块的「规则」字段关联到站点。

### 规则类型

#### WAF（Web 应用防火墙）

使用 ModSecurity SecLang 语法检测 Web 攻击（SQL 注入、XSS、命令注入等）。

**表达式格式**：每行一条 SecLang 指令

```
SecRule ARGS "@detectSQLi" "id:'1',phase:2,deny,status:403,msg:'SQL Injection'"
```

**默认模板**：
- XML 请求体解析器
- JSON 请求体解析器
- 请求体验证规则

#### IP（IP 地址规则）

基于客户端 IP 地址的黑/白名单。

**表达式格式**：

| 字段 | 说明 |
|------|------|
| 名称 | 规则描述（如 "内网地址"） |
| 运算符 | `属于` 或 `不属于` |
| IP 列表 | 逗号分隔的 IP 或 CIDR（如 `10.0.0.0/8, 192.168.0.0/16`） |

**默认模板**：
- loopback: 属于 127.0.0.1
- lan cidr: 属于 10.0.0.0/8, 192.168.0.0/16

#### User-Agent（UA 匹配）

基于请求 User-Agent 头的过滤规则。

**运算符**：

| 运算符 | 说明 |
|--------|------|
| 等于 | 完全匹配 |
| 不等于 | 不等于指定值 |
| 包含 | 包含子字符串 |
| 不包含 | 不包含子字符串 |
| 正则匹配 | 正则表达式匹配 |

**默认模板**：当前浏览器的 User-Agent

#### IP 限速（IP Rate Limiting）

限制每个 IP 的请求频率，超限后封禁一段时间。

**配置项**：

| 字段 | 说明 |
|------|------|
| 速率 | 每个 IP 每秒允许的最大请求数（默认 100） |
| 封禁时长 | 超限后封禁秒数（默认 6000 秒 ≈ 100 分钟） |

#### 组合规则（Compound）

将多条已有规则通过逻辑运算符组合。

**逻辑运算符**：
- `begin`：第一条规则（起始点）
- `AND`：两条规则都匹配时触发
- `OR`：任一规则匹配时触发

**示例**：
```
begin: ip-blacklist-rule
AND:   ua-bot-rule
```
含义：IP 在黑名单中 **且** User-Agent 是机器人时才阻止。

### 通用字段

| 字段 | 说明 |
|------|------|
| 类型 | WAF / IP / User-Agent / IP 限速 / 组合规则 |
| 操作 | 允许 / 阻止（WAF 类型由 SecLang 内部定义，不显示此字段） |
| 状态码 | 匹配时返回的 HTTP 状态码（如 403、400） |
| 原因 | 显示给被拦截用户的错误信息 |
| 详细模式 | 开启后，错误页显示匹配的规则名和表达式详情 |

### 操作指南

1. 进入 **LLM AI → 规则**，点击「添加规则」
2. 选择规则类型 — 表达式编辑区域会自动切换为对应类型的专用编辑器
3. 添加表达式（可点击「恢复默认」加载预设模板）
4. 设置操作（允许/阻止）、状态码和原因
5. 保存规则
6. 进入 **站点管理**，在目标站点的「规则」区域添加此规则

## 模块间关系

```
MCP 商店 ──(一键安装)──→ MCP 服务器 ──(同步)──→ 工具列表
                              ↑
                         AI Agent 通过代理调用
                              
智能体 ──(关联应用)──→ Application

站点 ──(关联规则)──→ 规则列表 ──(按顺序执行)──→ 允许/阻止
  ↓
反向代理 → 后端服务

入口 ←──(自动写入)── 日志提供者 (权限日志 / OpenClaw / 采集器)
```

## 与授权系统的集成

LLM AI 模块在三个层面与 JetAuth 授权系统结合：

| 层面 | 机制 | 说明 |
|------|------|------|
| API 访问控制 | Casbin 策略 | 所有 `/api/*` 请求经过 AuthZ Filter，控制谁能管理这些资源 |
| MCP 工具级权限 | Tool.IsAllowed | MCP 代理转发前检查每个工具的允许状态 |
| MCP Self JWT Scope | OAuth2 Scope → Tool 映射 | JetAuth 自身作为 MCP 服务器时，通过 JWT token 中的 scope 控制工具访问 |
| 站点网关层 | Rule 规则引擎 | 独立于 Casbin 的 WAF/IP/UA/限速规则引擎 |
